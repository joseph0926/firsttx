## 운영 리스크

- 이슈 해결 속도가 빠르면 `Last Verified`가 빠르게 stale 될 수 있습니다.
- `README` 인덱스와 영역 문서를 동시에 갱신하지 않으면 상태 드리프트가 재발합니다.
- 테스트 공백(`TC-*`)이 해소되지 않으면 보안/성능 수정의 회귀 탐지력이 부족할 수 있습니다.
